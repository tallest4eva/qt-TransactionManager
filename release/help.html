Transaction Manager Overview
The Transaction Manager is an application which aggregates, manages and displays financial transactions contained in a data file.
It allows users to view the contents of a configurable input data file containing account and transaction information.



Release Contents
The Transaction Manager 1.09 release contains:
  • TransactionManager.exe --- Application executable
  • help.html and help/ directory --- For help documentation.
  • Transactions - Test.csv --- Test input data file
  • config.xml --- Saved Config file (with predefined loaded configs, categories, and labels)
  • Qt dlls
  • MingGW dlls
Attribution:
  • Qt 5.9 --- For HMI Framework
  • QCustom Plot --- For graph plots
  • PieView --- For pie chart plots.

How To Use

Open Transaction Data File
  • Click on the 'File' menu
  • Click on the 'Open' menu item


  • Select transaction data file to display transactions
    • NOTE: The transaction data file is never modified by this application.
  • The file name of the transaction file being reviewed is displayed in the bottom-left corner of the status bar



Transaction Manager Display
The Transaction Manager displays transaction data in 3 general tabs:

The Overview Tab

The Overview tab displays a high level overview of the information
  • An Accounts list to display all the accounts found
    • Displays the Name, Current Balance, Transaction Count, Status, Open and Last Date (Close Date if closed) of each account found
    Double-Click on an account item in the Accounts list to display the selected account's transactions in the Transactions tab.

  • The Number of Accounts found
  • The Number of Transactions found
  • The combined current Net Worth of the accounts found
  • The Date of the First and Last transactions
  • A Check Box to Show or Hide Closed accounts in the accounts list.
  • A Drop Down Box to Sort the accounts list by "Account Name", "Balance" and "Open Date".

The Transactions Tab
The Transactions tab displays the transactions in a customizable list.
It displays the Date, Account Name, Description, AMount, Balance, Category and Labels for each transaction displayed.

  • The Transactions list is, by default, sorted by date from most recent to oldest transaction.
  • The Transactions list can sorted by any of the transaction values by clicking the desired header.
  • The Transaction selection tool box can be used to filter the transactions list's contents. See Filter Selection Toolbox
  • The 'Apply' button updates the transaction list with the current filter
  • The 'Show All' button clears all transaction filters and displays entire transaction list
  • The 'Show Filter In Reports' button displays the current transaction filter in the Reports tab.
  • The 'Filter Transactions' text box allows the user to filter the displayed transactions in the transactions list
  • The 'Total Credit', 'Total Debit' and 'Net Amount' labels display the total amounts on the transaction list

The Reports Tab
The Reports tab contains mutiple tabs containing the Report charts for the selected data.

  • The Report selection tool box can be used to filter the items displayed in the reports. See Filter Selection Toolbox
  • The 'Apply' button updates the report tabs with the current filter
  • The 'Show All' button clears all report filters and displays entire reports
  • The 'Show Filter In Transations' button displays the current report filter in the Transactions tab.
The 'Net Income' Reports Tab
The 'Net Income' report tab displays a bar chart of the cumulative monthly income (green bars) and expense (red bars) over time.

The 'Net Worth' Reports Tab
The 'Net Worth' report tab displays a bar chart of the cumulative monthly net worth (green bars) over time.

  • Click on a bar on the bar chart to display popup information on the month selected.


  • Double-Click on the popup to display the selected month's transactions in the Transactions tab.
  • Click on the bar chart to hide the popup.
  • Click and drag across the bar chart to select a date interval to view in the Reports tab.


The 'Report List' Reports Tab
The 'Report List' report tab displays a table of the cumulative monthly income, expense, net income and net worth over time.
The last row displays the totals of the cumulative income, expense, net income, net transfers, and net worth.

  • Double-Click on a row to display the selected month's transactions in the Transactions tab.
  • Double-Click on an Income, Expense, Net Income and Transfers item to display the selected month's transactions in the Transactions tab filtered byt the respective categories.
The 'Account Balance' Reports Tab
The 'Account Balance' report tab displays a graph of the individual account balances over time.
The last row displays the totals of the cumulative income, expense, net income, net transfers, and net worth.

  • Click on the graph to display infomation on the selected and top account balances for the selected time period.
The 'Assets/Debts by Account' Reports Tab
Displays pie charts of the Balances of the Accounts at the last date.
Accounts with positive balances are displayed in the 'Assets by Account' pie chart.
Accounts with negative balances are displayed in the 'Debts by Account' pie chart.
Use the Drop Down Box to sort the pie chart "By Balance" or "By Name".

The 'Income/Expense by Parent Category' Reports Tab
Displays pie charts of the cumulative Income and Expense for parent categories over the selected date interval.
Income parent categories are displayed in the 'Income by Parent Category' pie chart.
Expense parent categories are displayed in the 'Expense by Parent Category' pie chart.
Use the Drop Down Box to sort the pie chart "By Balance" or "By Name".

The 'Income/Expense by Sub Category' Reports Tab
Displays pie charts of the cumulative Income and Expense for sub categories over the selected date interval.
Income sub categories are displayed in the 'Income by Sub Category' pie chart.
Expense sub categories are displayed in the 'Expense by Sub Category' pie chart.
Use the Drop Down Box to sort the pie chart "By Balance" or "By Name".

  • Click on an item on the pie chart/key to display popup information about the Account or Category Selected.


  • Double-Click on the popup to display the selected Account or Category's transactions in the Transaction Tab.
  • Click on the pie chart to hide the Popup.

The Filter Selection Tool Box
The Filter Selection Toolbox allows users to create customized filters to apply to the Transactions and Reports tabs.
The Filter Selection Toolbox is enabled when the user opens an input data file.
  • The 'Show All' button clears all selection filters and displays all data available.
  • The 'Show Filter In Transactions/Reports' button displays the current filter in the Transactions or Reports tab.
  • The 'Accounts' toolbox is used to select the accounts to be viewed.
    • The 'Accounts' toolbox contains a checkbox for each account found.
    The 'Accounts' toolbox title updates to 'All' if all the accounts are selected, and "Some" if any account is being filtered out.
    Check/Uncheck the 'All Accounts' checkbox to check/uncheck every account.
    Check/Uncheck the 'Open Accounts' checkbox to check/uncheck every open account.


  • The 'Categories' toolbox is used to select the categories to be viewed.
    • The 'Categories' toolbox contains a tree of every category supported.
    Use the 'Expand All'/'Collapse All' button for toggling between expanding and collapsing the 'Categories' tree.
    Check/Uncheck the Parent Categories tree checkbox to select/unselect all it's sub categories.
    Check/Uncheck the 'All Categories' checkbox to check/uncheck every category.
    Check/Uncheck the 'Income Categories' checkbox to check/uncheck every income category.
    Check/Uncheck the 'Expense Categories' checkbox to check/uncheck every expense category.
    The 'Categories' toolbox title updates to 'All' if all the categories are selected, and "Some" if any category is being filtered out.
    Unused categories checkboxes are always disabled.


  • The 'Labels' toolbox is used to select the labels to be viewed.
    • The 'Labels' toolbox contains a entry for each label supported.
    The 'Labels' entries allow the user to 'Match', 'Don't Match' or 'Exclude' each label.
    The 'Labels' toolbox title updates to 'All' if all the labels are matched, and "Some" if any label is being filtered.
    Unused labels buttons are always disabled.
    Check/Uncheck the 'All Labels' checkbox to select/unselect every label.


  • The 'Date Interval' toolbox is used to select the date interval for the filter.
    • The 'Date Interval' toolbox contains a combo box to select the date interval to view.

    Selecting the 'All Time' dropdown updates the filter to the maximum date interval and applies the filter.
    Selecting the 'This Year' dropdown updates the start date to the beginning of the current year and applies the filter.
    Selecting the 'Last 1 Year' dropdown updates the start date to 1 year ago and applies the filter.
    Selecting the 'Last 3 Years' dropdown updates the start date to 3 years in the past and applies the filter.
    Selecting the 'Last 5 Years' dropdown updates the start date to 5 years in the past and applies the filter.
    Selecting the 'Last 10 Years' dropdown updates the start date to 10 years in the past and applies the filter.
    The 'Date Interval' toolbox title updates to 'All' if the maximum date interval is selected, and 'Some' if the date interval selected is less than the maximum date interval.
    Selecting the 'Custom' dropdown enables the 'Start Date' and 'End Date' edit boxes which can be used to modify the date interval viewed.

    Unique to the Report tab, the 'Report Period' combo box is used to set what report period is displayed. (Monthly, Quarterly, Biannually or Annually)

    Unique to the Report tab, the 'Show Net Income' check box is used to toggle the 'Net Income' plot displayed on the 'Income / Expense' graph.

    Unique to the Report tab, the 'Show Trasfers' check box is used to toggle whether to display Transfers in 'Net Income' graph, 'Income/Expense' and 'Sub-Income/Sub-Expense' charts.

  • The 'Apply' button applies the selected filter to the Transactions or Reports tabs.

Input Data File Description
The Transaction Manager takes a configurable text data file as input.

Display Input File Contents
You can view the contents of the input file being viewed.
  • Press on the 'Tools' Menu
  • Press on the 'Display File Contents' Menu Item


  • This opens the 'Display File Contents' Dialog


  • Press the 'Done' button to close dialog.

Default Input File Format
The default input data file is a csv (comma-separated-values) file.
The csv file contains lines describing accounts and transactions.
All other lines are ignored.
NOTE: As per csv guildelines, Fields containing a comma should be quoted.

Transaction Descriptor Line:
Transaction,12/10/12,Bank - Credit Card-2000,QuikTrip,CREDITCARD QT 200,debit,-80.00,660.00,Gas & Fuel,Reviewed Manual
NOTE: Transactions with unmatched category labels are defaulted with a red 'Uncategorized' category in the Transactions tab.

NOTE: If the current balance is NOT provided for a transaction in an Incomplete account, its current balance shows up as a red 'Unknown' in the Transaction tab.

  • The 1st column must have the 'Transaction' keyword.
  • The 2nd column contains the transaction Date. Default date format: 'MM/dd/yy'
  • The 3rd column contains the transaction's Account Name.
  • The 4th column contains the transaction's Description.
  • The 5th column contains the transaction's Original Description. (Optional)
  • The 6th column contains the transaction's Type. (Optional)
    • NOTE: If the transaction's Type is omitted, it is determined by the transaction's Amount
    Positive and Negative amounts are determined as 'credit' and 'debit' respectively.
    Valid transaction Type values
    debit
    credit
  • The 7th column contains the transaction's Amount.
  • The 8th column contains the transaction account's Current Balance. (Optional)
    • NOTE: If the transaction account's state is Complete and the Current Balance is omitted, it is automatically calculated.
  • The 9th column contains the transaction's Category.
    • Valid transaction parent categories

'Input File Config' Dialog
The format of the input data file can be configured by the 'Input File Config' Dialog.
  • Press on the 'Tools' Menu
  • Press on the 'Configure Input File' Menu Item


  • This opens the 'Input File Config' Dialog
    • This allows the user to select the current preset configuration.
    The current preset config is always displayed in the bottom-right corner of the status bar.

  • Config Presets
    • Default Config
      • The default configuration is not configurable.


    • Custom Config
      • This allows the user to configure custom configs for the input data file.

    • Use the checkbox, dropdown and text edit boxes to configure:
      • File Separator: Sets the file separator character. Valid values: ',' and '|'
      • Date Format: Sets the date format used for date fields. Default: 'M/d/yy'
      • The Account line column 1 keyword. Default: 'Account'
      • The Account descriptor column order
      • Use Transaction Keyword: Sets whether Transaction Descriptor Keywords are used.
        • NOTE: Unchecking this disables the Transaction keyword line edit box.
      • The Transaction line column 1 keyword. Default: 'Account'
      • The Transaction descriptor column order
    • Press the 'Cancel' button to close the dialog without saving the configuration.
    • Press the 'Done' button to save the selected configuration.

    • Loaded Configs (Advanced Users)
      • This allows the user to load saved configurations from a config file: config.xml.
      Loaded configs are displayed if a config file is located in the applications directory.
      For convenience, a config file is provided in the release with preset configs for several common finanical export files: Mint, Bank of America, Chase Bank and TD Ameritrade.
      The config file also contains the full sub category definitions listed above.

      The config file is a simple xml file which adds the ability to add config descriptions and add new categories.
      Sample Config file content:
      <configs\>
        <!-- Config definitions -->
        <config name='Sample Config'>
          <dateFormat></dateFormat>
          <account keyword='' name='' status='' state='' alternateNames='' ></account>
          <transaction keyword='' date='' name='' description='' originalDescription='' type='' amount='' balance='' category='' labels='' ></transaction>
        </config>
        <!-- Category definitions -->
        <category category='Parent Category' subCategory='Sub Category'></category>
      </configs>
        <!-- Labels definitions -->
        <label name='SampleLabel'></category>
      </configs>

      Config Node Description:
      • The 'name' attribute contains the Config Name.
        • NOTE: If the config name is 'Default Config', it replaces the default config for the application.
      • The 'dateFormat' node contains the Date Format. Default: 'M/d/yy'
      • The 'account' node contains information about the account descriptor line.
        • NOTE: Do not include 'account' node if the config does not use account descriptor lines.
        • The 'keyword' attribute specifies the account descriptor line keyword
        • The 'name', 'status', 'state', and 'AlternateNames' attributes specify the account descriptor line column numbers.
          • NOTE: Attributes for unavailable columns are left empty or omitted.
      • The 'transaction' node contains information about the transaction descriptor line.
        • The 'keyword' attribute specifies the transaction descriptor line keyword
          • NOTE: If not using a transaction descriptor keyword, the keyword attribute is left empty or omitted.
        • The 'date', 'name', 'description', 'originalDescription', 'type', 'amount', 'balance', 'category', 'labels' attributes specify the transaction descriptor line column numbers.
          • NOTE: Attributes for unavailable columns are left empty or omitted.
      Category Node Description:
      • The 'category' attribute contains the Parent Category.
        • NOTE: This should match an existing Parent Category. Defaults to "Uncategorized" Category.
      • The 'subCategory' attribute contains the new Sub Category string.
      Label Node Description:
      • The 'name' attribute contains the new Labels tag (No spaces allowed).
