Transaction Manager Overview
The Transaction Manager is an application which aggregates, manages and displays financial transactions contained in a data file.
It allows users to view the contents of a configurable input data file containing account and transaction information.



How To Use

Open Transaction Data File
  • Click on the 'File' menu
  • Click on the 'Open' menu item

  • Select transaction data file to display transactions
  • The file name of the transaction file being reviewed is displayed in the status bar


Transaction Manager Display
The Transaction Manager displays transaction data in 3 general tabs:

The Overview Tab

The Overview tab displays a high level overview of the information
  • An Accounts list to display all the accounts found
    • Displays the Name, Current Balance, Status, Open and Close Date(if closed) of each account found
    Double-Click on an account item in the Accounts list to display the selected account's transactions in the Transactions tab.

  • The Number of Accounts found
  • The Number of Transactions found
  • The combined current Net Worth of the accounts found
  • The Date of the First and Last transactions
The Transactions Tab
The Transactions tab displays the transactions in a customizable list

  • The Transactions list is, by default, sorted by date from most recent to oldest transaction.
  • The Transactions list can sorted by any of the transaction values by clicking the desired header.
  • The Transaction selection tool box can be used to filter the transactions list's contents. See Filter Selection Toolbox
  • The 'Apply' button updates the transaction list with the current filter
  • The 'Show All' button clears all transaction filters and displays entire transaction list
The Reports Tab
The Reports tab contains mutiple tabs containing the Report charts for the selected data.

  • The Report selection tool box can be used to filter the items displayed in the reports. See Filter Selection Toolbox
  • The 'Apply' button updates the report tabs with the current filter
  • The 'Show All' button clears all report filters and displays entire reports
The 'Net Income' Reports Tab
The 'Net Income' report tab displays a bar chart of the cumulative monthly income (green bars) and expense (red bars) over time.

The 'Net Worth' Reports Tab
The 'Net Worth' report tab displays a bar chart of the cumulative monthly net worth (green bars) over time.

  • Click on a bar on the bar chart to display popup information on the month selected.

  • Double-Click on the popup to display the selected month's transactions in the Transactions tab.
  • Click on the bar chart to hide the popup.
  • Click and drag across the bar chart to select a date interval to view in the Reports tab.

The 'Report List' Reports Tab
The 'Report List' report tab displays a table of the cumulative monthly income, expense, net income and net worth over time.
The last row displays the totals of the cumulative income, expense, net income and net worth.

  • Double-Click on a row to display the selected month's transactions in the Transactions tab.
The 'Assets/Debts by Account' Reports Tab
Displays pie charts of the Balances of the Accounts at the last date.
Accounts with positive balances are displayed in the 'Assets by Account' pie chart.
Accounts with negative balances are displayed in the 'Debts by Account' pie chart.

The 'Income/Expense by Parent Category' Reports Tab
Displays pie charts of the cumulative Income and Expense for parent categories over the selected date interval.
Income parent categories are displayed in the 'Income by Parent Category' pie chart.
Expense parent categories are displayed in the 'Expense by Parent Category' pie chart.

The 'Income/Expense by Sub Category' Reports Tab
Displays pie charts of the cumulative Income and Expense for sub categories over the selected date interval.
Income sub categories are displayed in the 'Income by Sub Category' pie chart.
Expense sub categories are displayed in the 'Expense by Sub Category' pie chart.

  • Click on an item on the pie chart/key to display popup information about the Account or Category Selected.

  • Double-Click on the popup to display the selected Account or Category's transactions in the Transaction Tab.
  • Click on the pie chart to hide the Popup.

The Filter Selection Tool Box
The Filter Selection Toolbox allows users to create customized filters to apply to the Transactions and Reports tabs.
The Filter Selection Toolbox is enabled when the user opens an input data file.
  • The 'Show All' button clears all selection filters and displays all data available.
  • The 'Accounts' toolbox is used to select the accounts to be viewed.
    • The 'Accounts' toolbox contains a checkbox for each account found.
    The 'Accounts' toolbox title updates to 'All' if all the accounts are selected, and "Some" if any account is being filtered out.
    Check/Uncheck the 'All Accounts' checkbox to check/uncheck every account.
    Check/Uncheck the 'Open Accounts' checkbox to check/uncheck every open account.


  • The 'Categories' toolbox is used to select the categories to be viewed.
    • The 'Categories' toolbox contains a checkbox for each category supported.
    The 'Categories' toolbox title updates to 'All' if all the categories are selected, and "Some" if any category is being filtered out.
    Unused categories checkboxes are always disabled.
    Check/Uncheck the 'All Categories' checkbox to check/uncheck every category.
    Check/Uncheck the 'Income Categories' checkbox to check/uncheck every income category.
    Check/Uncheck the 'Expense Categories' checkbox to check/uncheck every expense category.


  • The 'Labels' toolbox is used to select the labels to be viewed.
    • The 'Labels' toolbox contains a checkbox for each label supported.
    The 'Labels' toolbox title updates to 'All' if all the labels are selected, and "Some" if any label is being filtered.
    Unused labels checkboxes are always disabled.
    Check/Uncheck the 'All Labels' checkbox to check/uncheck every label.


  • The 'Date Interval' toolbox is used to select the date interval for the filter.
    • The 'Date Interval' toolbox contains a 'Start Date' and an 'End Date' edit box which can be used to modify the date interval viewed.
    The 'This Year' button updates the start date to the beginning of the current year and applies the filter.
    The 'Last 1 Year' button updates the start date to 1 year ago and applies the filter.
    The 'Last 5 Years' button updates the start date to 5 years in the past and applies the filter.
    The 'Last 10 Years' button updates the start date to 10 years in the past and applies the filter.
    The 'All Time' button updates the filter to the maximum date interval and applies the filter.
    The 'Date Interval' toolbox title updates to 'All' if the maximum date interval is selected, and 'Some' if the date interval selected is less than the maximum date interval.


  • The 'Apply' button applies the selected filter to the Transactions or Reports tabs.

Input Data File Description
The Transaction Manager takes a configurable text data file as input.

Default Input File Format
The default input data file is a csv (comma-separated-values) file.
The csv file contains lines describing accounts and transactions.
All other lines are ignored.

Account Descriptor:
Account,,Bank - Checking Account-0002,Open,Complete,Bank - Checking Account-0001;Bank - Checking Account-0002
  • The 1st column must have the 'Account' keyword.
  • The 2nd column is left empty.
  • The 3rd column contains the Account Name.
  • The 4th column contains the Account Status stating whether the account is opened or close.
    • Valid Account Status values
    Open
    Closed
  • The 5th column contains the Account State stating whether the account has all its transactions present in the data file.
    • Valid Account State values
    Complete
    Incomplete
     NOTE: If an account is marked as complete, transactions without current balances are calculated automatically.
  • The 6th column contains a semi-colon separated list of Alternate Account Names. (Optional)
Transaction Descriptor:
Transaction,12/10/12,Bank - Credit Card-2000,QuikTrip,CREDITCARD QT 200,debit,-80.00,660.00,Gas & Fuel,Reviewed Manual
NOTE: Transactions with unmatched account labels are defaulted with a red 'None' account name in the Transactions tab.
Transactions with unmatched category labels are defaulted with a red 'Uncategorized' category in the Transactions tab.

NOTE: If the current balance is NOT provided for a transaction in an Incomplete account, its current balance shows up as a red 'Unknown' in the Transaction tab.

  • The 1st column must have the 'Transaction' keyword.
  • The 2nd column contains the transaction Date. Default date format: 'MM/dd/yy'
  • The 3rd column contains the transaction's Account Name.
  • The 4th column contains the transaction's Description.
  • The 5th column contains the transaction's Original Description. (Optional)
  • The 6th column contains the transaction's Type.
    • Valid transaction Type values
    debit
    credit
  • The 7th column contains the transaction's Amount.
  • The 8th column contains the transaction account's Current Balance. (Optional if the account state is Complete)
  • The 9th column contains the transaction's Category.

Display Input File Contents
You can view the contents of the input file being viewed.
  • Press on the 'Tools' Menu
  • Press on the 'Display File Contents' Menu Item

  • This opens the 'Display File Contents' Dialog

  • Press the 'Done' button to close dialog.

'Input File Config' Dialog
The format of the input data file can be configured by the 'Input File Config' Dialog.
  • Press on the 'Tools' Menu
  • Press on the 'Configure Input File' Menu Item

  • This opens the 'Input File Config' Dialog

  • Use the dropdown and text edit boxes to configure:
    • The file separator character. Valid values: ',' and ';'. Default: ','
    • The date format used for date fields. Default: 'MM/dd/yy'
    • The Account line column 1 keyword. Default: 'Account'
    • The Account descriptor column order
    • The Transaction line column 1 keyword. Default: 'Account'
    • The Transaction descriptor column order
  • Press the 'Restore Default' button to restore the default input file configuration.
  • Press the 'Done' button to save the configuration or 'Cancel' to close without saving.
